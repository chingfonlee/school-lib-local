# 安全政策

## 資料安全

本系統設計為本機部署，預設僅綁定 `127.0.0.1`（本機），不對外開放連線。

**請勿提交以下內容至版本控制：**

- `config.yaml`（含密碼與 session secret）
- `data/`（資料庫，含館藏與採購記錄）
- `exports/`（匯出的採購書單 Excel 檔）
- `00_source/`（書商 Excel 原始書單）
- 任何含有學校名稱、師生個人資料的檔案

上述項目均已列入 `.gitignore`，請確認未誤用 `git add -f` 或 `git add .` 強制加入追蹤。

## 設定安全

首次安裝後請立即執行以下步驟：

1. 將 `config.yaml` 的 `default_admin_password` 改為強密碼
2. 將 `session_secret_key` 改為隨機長字串（至少 32 字元）

產生隨機 secret 的方式：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

若未修改預設值，系統安全性較低，不建議在多人共用網路環境下使用。

## 部署安全

本系統預設綁定 `127.0.0.1:8765`，僅供本機使用。若需在區域網路或公開網路存取，請自行評估並強化部署安全（反向代理、防火牆、HTTPS 等）。本專案不提供此類設定的官方支援。

## 回報安全問題

發現安全漏洞或疑似敏感資料外洩的問題，請透過 GitHub Issue 或私下聯絡維護者，避免在公開 issue 中直接揭露漏洞細節。

> 聯絡方式與回應時程可由維護者進一步說明與更新。
