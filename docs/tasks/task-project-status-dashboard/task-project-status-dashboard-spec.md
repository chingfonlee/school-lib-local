# Spec: task-project-status-dashboard

## 目前問題

`index.html` 已有 5 個統計卡片（館藏 / 書商書單 / 可採購 / 已選書 / 上次匯出），但：

- 只顯示原始數字，不根據專案狀態給出任何判斷
- 使用者不知道「下一步要做什麼」
- 有已選書時，不顯示匯出準備度（就緒 / 待補充 / 缺必填筆數）

## 改善目標

在首頁統計卡片下方加入「下一步建議」卡片：

- 根據當前專案狀態，顯示最重要的下一步動作與連結
- 有已選書時，顯示匯出準備度摘要（就緒 / 待補充 / 缺必填各幾本）
- 不引入圖表或大型 dashboard，只做狀態判斷 + 文字導引

## 下一步建議邏輯

| 條件（依序判斷） | 顯示文字 | 連結 |
|-----------------|---------|------|
| 館藏筆數 = 0 | 尚未匯入館藏，建議先匯入館藏再繼續 | 匯入頁面 |
| 書商書單筆數 = 0 | 尚未匯入書商書單 | 匯入頁面 |
| 可採購筆數 = 0（書單已匯入） | 書商書單比對後無可採購書籍，請確認館藏內容是否完整 | 比對結果頁面 |
| 已選書 = 0（有可採購） | 有 N 本可採購，建議前往選書 | 選書頁面 |
| 已選書 > 0 | 已選 N 本 — 就緒 A 本、待補充 B 本、缺必填 C 本 | 匯出前檢查頁面 |

## 資料來源

所有資料均使用既有 API，不新增 endpoint：

| 資料 | API | 備註 |
|------|-----|------|
| 館藏筆數 | `/api/books/stats` | 已呼叫 |
| 書商書單筆數 | `/api/books/stats` | 已呼叫 |
| 可採購筆數 | `/api/books/stats` | 已呼叫 |
| 已選書數量 | `/api/selections/` | 已呼叫 |
| 匯出準備度 | `/api/exports/check` | 已存在，條件呼叫（已選書 > 0 時） |

## 不做的事

- 不引入圖表套件（Chart.js 等）
- 不新增後端 API endpoint
- 不改資料庫 schema
- 不做預算追蹤或警示通知
- 不做複雜多欄 dashboard
- 不做多使用者或角色權限差異
- 不改既有統計卡片的顯示

## 驗收條件

1. 館藏筆數 = 0 時，下一步建議顯示「尚未匯入館藏」與匯入連結
2. 有館藏、書商書單 = 0 時，顯示「尚未匯入書商書單」與匯入連結
3. 有書單、可採購 = 0 時，顯示「無可採購書籍」與比對結果連結
4. 有可採購、已選書 = 0 時，顯示「有 N 本可採購，建議前往選書」與選書連結
5. 已選書 > 0 時，顯示就緒 / 待補充 / 缺必填筆數與匯出前檢查連結
6. 統計卡片既有顯示不受影響
7. `python -m compileall app` 通過
8. `python -m pytest -v` 通過（既有測試全部通過）
9. 手動驗證：各狀態情境顯示正確
