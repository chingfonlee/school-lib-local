# Spec: task-import-flow-guidance

## 目前問題

匯入流程功能已完整，但成功與錯誤後的使用者導引偏弱：

- 館藏匯入成功後：顯示筆數與比對資訊，但**無下一步連結**，使用者不知道該前往哪裡
- 書商書單匯入成功後：有「前往比對結果」連結，但**缺「前往選書」**第二條導引路徑
- 錯誤訊息：直接顯示後端 `str(e)` 技術錯誤，**無欄位對應提示**，使用者無從判斷如何修正
- 單一工作表館藏檔案匯入後：不顯示工作表名稱，摘要資訊較少

## 改善目標

讓教師 / 學校行政人員在匯入後：
- 清楚知道匯入是否成功、匯入了多少資料
- 看到明確的下一步導引
- 遇到錯誤時，能收到白話說明與欄位對應提示，而非 Python 錯誤文字

## 需求範圍

### 1. 館藏匯入成功訊息改善（`uploadHoldings()` 成功段落）

新增：
- 無論單一或多個工作表，均顯示各工作表筆數摘要
- 下一步連結：「前往比對結果 →」（`/match.html`）

### 2. 書商書單匯入成功訊息改善（`doConfirmImport()` 成功段落）

新增：
- 在現有「前往比對結果 →」旁加入「前往選書 →」（`/selection.html`）

### 3. 錯誤訊息白話化（前端 helper）

新增 `formatImportError(message)` helper function：
- 若訊息已是中文、使用者可理解的說明（如 `館藏檔案未匯入任何有效資料...`），直接保留
- 若訊息為技術性 Python 錯誤（含英文 exception 關鍵字或過長），改為通用白話提示：
  - 館藏：「匯入失敗。請確認 Excel 格式正確、欄位含有 ISBN 或書名。」
  - 書商書單（步驟 A 讀取預覽）：「無法讀取檔案，請確認為有效的 .xlsx 格式。」
  - 書商書單（步驟 E 確認匯入）：「匯入失敗。請確認欄位對應正確，書名、ISBN 欄位有資料。」
- 不暴露 Python traceback 或 exception class 名稱

### 4. 首頁 / import 頁面專案狀態顯示

**本 task 不實作。** 需要新增 API 查詢館藏是否已匯入、書商書單筆數、已選書數量，評估後超出本 task 範圍。可留待後續任務。

## 不做的事

- 不重做整個流程為 wizard
- 不改資料庫 schema
- 不新增大型 dashboard
- 不引入前端框架
- 不修改匯入核心邏輯（import_service.py）
- 不修改 Excel 解析規則
- 不新增後端 API endpoint
- 不做 Playwright E2E
- 不做權限或多人協作
- 不在首頁顯示專案狀態

## 驗收條件

1. 館藏匯入成功後，顯示各工作表筆數摘要（含單一工作表）
2. 館藏匯入成功後，顯示「前往比對結果」連結
3. 書商書單匯入成功後，同時顯示「前往比對結果」與「前往選書」兩個連結
4. 館藏與書商書單匯入失敗時，錯誤訊息為白話中文，不含 Python traceback 或 exception 關鍵字
5. 既有匯入功能不受影響（local_culture 與 general_books 專案均可正常匯入）
6. `python -m compileall app` 通過（本 task 僅改 import.html，仍要確認 app 無損）
7. `python -m pytest -v` 通過（既有測試不受影響）
8. 手動驗證：館藏匯入、書商書單匯入、錯誤情境均通過
