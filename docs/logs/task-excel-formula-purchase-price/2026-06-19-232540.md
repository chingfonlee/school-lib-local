## 做了什麼

修正 vendor_books 匯入流程中，Excel 公式欄位（如 `=ROUND(G6*75%,0)`）因 openpyxl `data_only=True`
讀不到快取值而導致 `purchase_price` 全為 null 的問題。

**新增三個 helper 函式（`app/services/import_service.py`）**

- `_parse_formula_multiplier(formula)` — regex 提取 `*N%` 的乘數（如 `75%` → `0.75`）
- `_resolve_formula_purchase_price(formula, list_price)` — 以 `list_price` 為基準計算採購價；
  含 `ROUND` 則取整；list_price 為 None 或公式不符則回傳 None
- `_build_formula_fallback(file_bytes, sheet, header_row, pp_src_col)` — 以 openpyxl
  `data_only=False` 讀公式字串；按標題行定位 purchase_price 欄；
  回傳 `{data_row_0based: formula_string}`（key 對齊 DataFrame `enum_idx`）；例外靜默捕捉

**修改 `confirm_import`（兩步驟匯入路徑）**

- `pd.read_excel` 前建 `pp_formula_map`（engine 為 openpyxl 且有對應欄位才建）
- 迴圈中 `purchase_price is None and list_price is not None` 時以公式 fallback 補值

**修改 `import_vendor_books`（舊版單步驟匯入路徑）**

- `for _, row in df.iterrows()` 改為 `enumerate(df.iterrows())`，補上 `enum_idx`
- `_detect_header_row` 取 header row，再建 `pp_formula_map`
- 迴圈中套用相同 fallback 邏輯
