## 做了什麼

修正匯出 xlsx 超過範本資料列數時，footer 區塊（備註列、填表人/教務主任/校長簽核列）的 merged cells 與 row height 未正確位移的問題。

**根本原因**：openpyxl 的 `insert_rows` 位移儲存格資料，但不更新 `ws.merged_cells.ranges` 行號，也不位移 `ws.row_dimensions`。`_unmerge_ranges_overlapping_rows` 因此誤刪仍停留在舊行號的 footer merged ranges。

**修正方式**（`app/services/export_service.py`）：
- 新增 `_pop_footer_merged_ranges(ws, insert_at)`：在 `insert_rows` 前擷取並移除所有 `min_row >= insert_at` 的合併範圍；跨越插入點的範圍不保存，由現有清理邏輯移除。
- 新增 `_push_footer_merged_ranges(ws, saved, extra_rows)`：`_copy_row_style` 後以 `+extra_rows` 位移重建。
- 在 `_extend_data_rows` 加入 `footer_heights` dict，保存並於重建後還原 footer 列高（openpyxl 確認不自動位移 row_dimensions）。

**測試**（`tests/test_export_row_extension.py`）：
- 更新既有測試：`A80:L80` 正確改為 `A83:L83`（修正前鎖住的是錯誤行為）。
- 新增 5 個測試：footer 範圍位移、多範圍、extra_rows=0 不變、跨越插入點保守移除、row height 位移確認。
